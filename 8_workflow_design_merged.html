Your goal is to start with an existing nanobody and iteratively add mutations to it to improve its binding to the newest variant of the SARS-CoV-2 spike protein, resulting in 24 modified nanobodies with one or more mutations. Please determine how to use ESM, AlphaFold-Multimer, and Rosetta in this iterative design process. An important constraint is that ESM can evaluate all potential mutations to a nanobody in 5 minutes while AlphaFold-Multimer takes 30 minutes per mutation and Rosetta takes five minutes per mutation. The whole iterative process should take no more than a few days to complete. Note that AlphaFold-Multimer must be run before Rosetta on each mutation since Rosetta requires the nanobody-antigen structure predicted by AlphaFold-Multimer. Additionally, note that ESM log-likelihood ratios are generally in the range of 5-10 (higher is better), AlphaFold-Multimer interface pLDDT confidence scores are generally in the range of 60-80 (higher is better), and Rosetta binding energy scores are generally in the range of -20 to -40 (lower is better).

As a reference, here are the agenda questions from those meetings, which must be answered here as well:

  1. In each iteration, what is the order of operations for evaluating mutations with ESM, AlphaFold-Multimer, and Rosetta?

  2. In each iteration, how many mutations (give a single number) will you evaluate with ESM, AlphaFold-Multimer, and Rosetta?

  3. At the end of each iteration, how will you weigh the ESM, AlphaFold-Multimer, and/or Rosetta scores (give a formula) to rank the nanobody mutations?

  4. At the end of each iteration, how many of the top-ranked mutations (give a single number) will you keep for the next round?

  5. How will you decide how many iterations of mutations to run?

  6. After all of the iterations are complete, how exactly (step-by-step) will you select the final set of 24 modified nanobodies from across the iterations for experimental validation?

Principal Investigator, please provide your response to the agenda.

Principal Investigator’s Response

After reviewing all five meeting summaries, I have synthesized the best components from each into a unified, efficient, and scientifically rigorous plan. Below is my final decision on the iterative nanobody design pipeline, addressing all six agenda questions.

1. Order of Operations per Iteration

ESM → AlphaFold-Multimer → Rosetta

  • Step 1: ESM (5 min total for all mutations) – Screen all candidate mutations to rapidly eliminate poor binders.
  • Step 2: AlphaFold-Multimer (30 min per mutation) – Run only on top ESM-ranked mutations to predict the nanobody-antigen structure and obtain interface pLDDT.
  • Step 3: Rosetta (5 min per mutation) – Run on the same mutations after AlphaFold-Multimer, using the predicted structure to compute binding energy.

Rationale: This order minimizes computational time by using the fastest tool first and respects the dependency of Rosetta on AlphaFold-Multimer output. This is consistent across all summaries.

2. Number of Mutations Evaluated per Tool per Iteration

  • ESM: 2,280 mutations (all possible single-point mutations for a ~120-residue nanobody: 120 × 19 = 2,280). This is feasible in 5 minutes.
  • AlphaFold-Multimer: 50 mutations (top-ranked by ESM). This keeps AlphaFold runtime to 50 × 30 min = 25 hours, manageable within a few days when parallelized.
  • Rosetta: 50 mutations (same set as AlphaFold-Multimer). This adds 50 × 5 min = 4.2 hours, also manageable.

Total per iteration: ~29.2 hours (can be reduced to ~2.5 hours with parallelization on a cluster). This number (50) is a compromise between the 20–100 range seen in summaries, chosen to maximize throughput while staying within time constraints.

Source: The 2,280 number comes from summaries 1 and 3 (full mutation space). The 50 number for AF and Rosetta is from summary 5 (top 50), which balances depth and time better than 20 (summary 4) or 100 (summary 3).

3. Formula to Rank Mutations at End of Each Iteration

Composite Score = 0.2 × ESM_norm + 0.5 × AF_norm + 0.3 × Rosetta_norm

Where:

  • ESM_norm = (ESM_LLR – 5) / (10 – 5) → range 0–1, higher better
  • AF_norm = (AF_pLDDT – 60) / (80 – 60) → range 0–1, higher better
  • Rosetta_norm = 1 – (Rosetta_score + 40) / ( –20 + 40 ) = ( –20 – Rosetta_score ) / 20 → range 0–1, higher better (since lower raw Rosetta score is better)

Weights: w_ESM = 0.2, w_AF = 0.5, w_Rosetta = 0.3. These weights prioritize AlphaFold-Multimer (most reliable structure-based predictor) while still incorporating ESM (fast filter) and Rosetta (energetic refinement). This weighting is adapted from summary 2, which provided the most thoughtful justification for unequal weights.

Source: The normalization formulas are from summaries 1, 2, and 4. The specific weights (0.2, 0.5, 0.3) are from summary 2, which I consider the best balance of accuracy and practicality.

4. Number of Top-Ranked Mutations Kept per Iteration

Keep 5 top-ranked mutations from each iteration to carry forward to the next round.

Rationale: Keeping 5 allows for diversity while limiting combinatorial explosion in subsequent iterations. This number is consistent across summaries 1, 2, and 4, and is a reasonable compromise between the 10 (summary 5) and 24 (summary 3) proposals.

5. Number of Iterations

Run 3 iterations of single-point mutations (i.e., add one mutation per iteration). After 3 iterations, we will have nanobodies with up to 3 mutations.

Stopping criteria:

  • If after 2 iterations the composite score plateaus (improvement <5% in top 5), stop early.
  • Otherwise, complete 3 iterations.
  • Maximum 3 iterations to stay within a few days of compute.

Rationale: Three iterations are supported by summaries 1, 2, 4, and 5. This yields a manageable number of candidates (5 single + 5 double + 5 triple = 15) and fits within the time budget (~3–4 days with parallelization). Summary 3 proposed only 2 iterations, but 3 iterations allow deeper exploration of the mutation landscape.

6. Final Selection of 24 Modified Nanobodies

Step-by-step:

  1. Compile all candidates from all iterations that passed all three filters (ESM, AlphaFold-Multimer, Rosetta). This includes:

    • Iteration 1: 5 single mutants (top 5)
    • Iteration 2: 5 double mutants (top 5 built on best single mutants)
    • Iteration 3: 5 triple mutants (top 5 built on best double mutants)
    • Additionally, include the next-best candidates from each iteration (e.g., ranks 6–10 from each iteration) to expand the pool to at least 30 candidates.

  2. Re-rank all candidates using the same composite score formula (Section 3), but now normalize across the entire combined set (global min-max normalization) to ensure fair comparison across different mutation lengths.

  3. Select the top 24 candidates based on the global composite score.

  4. Diversity check: Ensure that the 24 candidates are structurally diverse (e.g., mutations in different CDR loops, different number of mutations). If two candidates are very similar (e.g., same mutation at same position), keep only the higher-scoring one and replace the duplicate with the next-ranked candidate.

  5. Validate structural feasibility: Manually inspect the top 24 AlphaFold structures for clashes or unusual geometry (using PyMOL or ChimeraX). Discard any with obvious issues and replace with the next-best candidate.

  6. Final list: Output 24 nanobody sequences (mutations relative to the parent) with their composite scores, ready for DNA synthesis and experimental binding assays.

Source: This step-by-step process combines the best elements from summaries 1 (global re-ranking, diversity check), 2 (balancing top scores with diversity), and 5 (structural validation). The inclusion of next-best candidates from each iteration is from summary 4.
